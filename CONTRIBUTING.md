## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue with:
- Clear description of the bug
- Steps to reproduce
- Expected vs actual behavior
- Screenshots (if applicable)
- Browser/OS information

### Suggesting Enhancements

Feel free to suggest new features by:
- Creating an issue with the "enhancement" label
- Describing the feature and its use case
- Explaining why it would be valuable

### Pull Requests

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Make your changes**
4. **Commit with conventional commits**
   ```bash
   git commit -m 'feat: add amazing feature'
   ```
5. **Push to your branch**
   ```bash
   git push origin feature/amazing-feature
   ```
6. **Open a Pull Request**

### Commit Message Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Code style changes (formatting, etc.)
- `refactor:` Code refactoring
- `perf:` Performance improvements
- `test:` Adding tests
- `chore:` Maintenance tasks
- `ci:` CI/CD changes

### Code Style

- Use ES6+ syntax
- Follow existing code formatting
- Write meaningful commit messages
- Keep components focused and reusable
- Add comments for complex logic

### Testing

Before submitting:
- Ensure the app builds without errors: `npm run build`
- Test in multiple browsers
- Check responsive design on mobile
- Verify dark mode works correctly
